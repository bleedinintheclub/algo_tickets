## Алгоритм Форда-Фалкерсона и Эдмондса-Карпа для поиска максимального потока

### Теория потоков

**Транспортная сеть** - орграф, в котором:
1. есть исток
2. есть сток
3. для каждого ребра задана пропускная способность `c(u, v) > 0`

**Поток** `f(u, v)` - величина, назначенная каждому ребру, такая что:
1. `0 <= f(u, v) <= c(u, v)`
2. `f(u, v) = - f(v, u)`
3. Для любой вершины, кроме стока и истока, втекающий поток равен вытекающему

**Величина потока** `|f|` - поток, выходящий из истока

**Остаточная сеть** - граф, состаящий из ребер, по которым еще можно пустить поток:
1. Прямое ребро имеет емкость `c(u, v) - f(u, v)`
2. Обратное ребро имеет емкость `f(u, v)` (сколько можно забрать назад)

**Теорема Форда-Фалкерсона**: величина максимального потока в транспортной сети равна минимальной пропускной способности разреза, разделяющего сток и исток.

**Разрез** — это разбиение вершин на две группы (S и T), где `s ∈ S`, а `t ∈ T`.

### Алгоритм Форда-Фалкерсона

#### Идея

Пока в остаточной сети есть путь от `s` до `t`, будем пускать по нему поток.

#### Принцип работы

1. Поток на всех ребрах равен 0.
2. Ищем любой увеличивающийся путь `(s, t)` (обычно через `DFS`)
3. Находим на пути ребро с минимальной пропускной способностью `c_min`
4. На всем пути `s, t` увеличиваем поток на `c_min` (и уменьшаем на обратных ребрах)
5. Повторяем, пока пути находятся

#### Асимптотика

Временная: `O(E * f)`, где `f` - максимальный поток
Пространственная: `O(V)`

#### Псевдокод

```python
# Находит увеличивающий путь и возвращает добавленный поток
def find_augmenting_path(u, target, min_cap):
    # Базовый случай: дошли до стока
    if u == target:
        return min_cap
    
    visited[u] = True
    
    # Пробуем пройти по всем соседям
    for v in adj[u]:
        # Остаточная пропускная способность ребра (u -> v)
        residual_cap = capacity[u][v] - flow[u][v]
        
        # Если в соседа еще не заходили и по ребру можно пустить поток
        if not visited[v] and residual_cap > 0:
            
            # Рекурсивно ищем путь дальше, обновляя минимальную емкость в пути
            delta = find_augmenting_path(v, target, min(min_cap, residual_cap))
            
            # Если путь до стока найден (delta > 0)
            if delta > 0:
                # Обновляем прямое ребро
                flow[u][v] += delta
                # Обновляем обратное ребро (уменьшаем поток в обратную сторону)
                flow[v][u] -= delta
                
                return delta
                
    return 0 # Путь не найден

# основной цикл
total_flow = 0
while True:
    visited = [False] * V
    # Ищем один увеличивающий путь с помощью DFS
    pushed = find_augmenting_path(source, sink, float('inf'))
    
    if pushed == 0: # Если путей больше нет — поток максимален
        break
        
    total_flow += pushed
```


#### Проблема алгоритма Форда-Фалкерсона

Классический Форд-Фалкерсон использует DFS для поиска пути. В худшем случае он может выбирать очень длинные пути с ничтожно малым потоком (например, по 1 единице), что приводит к сложности `O(E⋅f)`. Если поток огромен, алгоритм будет работать непозволительно долго.

### Алогритм Эдмонса-Карпа

#### Идея алгоритма 

Используем `BFS` вместо `DFS`. На каждой итерации выбираем кратчайший увеличивающий путь в остаточной сети. 

#### Принцип работы

1. Запускаем `BFS` из истока.
2. Идем только по ребрам, у которых `c(u, v) - f(u, v) > 0`
3. Останавливаемся, если достигли стока
4. Запоминаем предков вершин через `parent[]` для восстановления
5. В найденном пути ищем ребро с минимальной остаточной емкостью `d`
6. Для всех прямых ребер пути `f(u, v) += d` и для обратных `f(u, v) -= d`
7. Повторяем, пока `BFS` находит кратчайший путь

#### Асимптотика

Временная: `O(V * E^2)`
Пространственная: `O(V)` для массива предков

#### Псевдокод

```python
def BFS(s, t, parent):
    visited = [False] * V
    queue = [s]
    visited[s] = True
    
    while queue:
        u = queue.pop(0)
        for v in adj[u]:
            # Если не посещали и есть остаточная мощность
            if not visited[v] and (capacity[u][v] - flow[u][v] > 0):
                parent[v] = u
                visited[v] = True
                queue.append(v)
                if v == t: return True
    return False

def edmonds_karp(s, t):
    max_flow = 0
    parent = [-1] * V
    
    # Пока BFS находит кратчайший путь
    while BFS(s, t, parent):
        # 1. Находим минимальную емкость в найденном пути
        path_flow = float("Inf")
        v = t
        while v != s:
            u = parent[v]
            path_flow = min(path_flow, capacity[u][v] - flow[u][v])
            v = u
            
        # 2. Обновляем потоки вдоль пути
        v = t
        while v != s:
            u = parent[v]
            flow[u][v] += path_flow # Прямое ребро
            flow[v][u] -= path_flow # Обратное ребро
            v = u
            
        max_flow += path_flow
        
    return max_flow
```
